  <div class="mw-parser-output">
   <p>A <b>formal system</b> is used to infer theorems from axioms according to a set of rules. These rules used to carry out the inference of theorems from axioms are known as the <b>logical calculus</b> of the formal system. A formal system is essentially an "<a href="/wiki/Axiomatic_system" title="Axiomatic system">axiomatic system</a>"<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup>. In 1921, David Hilbert proposed to use such system as the foundation for the knowledge in mathematics <sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup>. A formal system may represent a well-defined <a href="/wiki/Abstraction" title="Abstraction">system of abstract thought</a>. Spinoza's <i><a href="/wiki/Ethics_(Spinoza)" title="Ethics (Spinoza)">Ethics</a></i> imitates the form of Euclid's <i>Elements</i>. Spinoza employed Euclidean elements such as "<a href="/wiki/Axiom" title="Axiom">axioms</a>" or "<a href="/wiki/Primitive_notion" title="Primitive notion">primitive truths</a>", <a href="/wiki/Rule_of_inference" title="Rule of inference">rules of inferences</a>, etc., so that a calculus can be built using these. </p>
   <p>Some theorists<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Avoid_weasel_words" class="mw-redirect" title="Wikipedia:Avoid weasel words"><span title="The material near this tag possibly uses too vague attribution or weasel words. (January 2017)">which?</span></a></i>]</sup> use the term <i>formalism</i> as a rough synonym for <i>formal system</i>, but the term is also used to refer to a particular style<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Avoid_weasel_words" class="mw-redirect" title="Wikipedia:Avoid weasel words"><span title="The material near this tag possibly uses too vague attribution or weasel words. (January 2017)">which?</span></a></i>]</sup> of <a href="/wiki/Notation" title="Notation">notation</a>, for example, <a href="/wiki/Paul_Dirac" title="Paul Dirac">Paul Dirac</a>'s <a href="/wiki/Bra%E2%80%93ket_notation" title="Bra–ket notation">bra–ket notation</a>. </p>

   <h2><span class="mw-headline" id="Background">Background</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Formal_system&amp;action=edit&amp;section=1" title="Edit section: Background">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>Each formal system uses primitive <a href="/wiki/Symbol_(formal)" title="Symbol (formal)">symbols</a> (which collectively form an <a href="/wiki/Alphabet_(computer_science)" class="mw-redirect" title="Alphabet (computer science)">alphabet</a>) to finitely construct a <a href="/wiki/Formal_language" title="Formal language">formal language</a> from a set of <a href="/wiki/Axiom" title="Axiom">axioms</a> through inferential <a href="/wiki/Rules_of_formation" class="mw-redirect" title="Rules of formation">rules of formation</a>. </p>
   <p>The system thus consists of valid formulas built up through finite combinations of the primitive symbols—combinations that are formed from the axioms in accordance with the stated rules.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> </p>
   <p>More formally, this can be expressed as the following: </p> 
   <ol>
    <li>A finite set of symbols, known as the alphabet, which concatenate formulas, so that a formula is just a finite string of symbols taken from the alphabet.</li> 
    <li>A <a href="/wiki/Formal_grammar" title="Formal grammar">grammar</a> consisting of rules to form formulas from simpler formulas. A formula is said to be <a href="/wiki/Well-formed_formula" title="Well-formed formula">well-formed</a> if it can be formed using the rules of the formal grammar. It is often required that there be a decision procedure for deciding whether a formula is well-formed.</li> 
    <li>A set of axioms, or <a href="/wiki/Axiom_schema" title="Axiom schema">axiom schemata</a>, consisting of well-formed formulas.</li> 
    <li>A set of <a href="/wiki/Rule_of_inference" title="Rule of inference">inference rules</a>. A well-formed formula that can be inferred from the axioms is known as a theorem of the formal system.</li>
   </ol> 
   <h3><span class="mw-headline" id="Recursive">Recursive</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Formal_system&amp;action=edit&amp;section=2" title="Edit section: Recursive">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p>A formal system is said to be <a href="/wiki/Recursive_set" title="Recursive set">recursive</a> (i.e. effective) or recursively enumerable if the set of axioms and the set of inference rules are <a href="/wiki/Decidable_set" class="mw-redirect" title="Decidable set">decidable sets</a> or <a href="/wiki/Recursively_enumerable_set" title="Recursively enumerable set">semidecidable sets</a>, respectively. </p> 
   <h3><span class="mw-headline" id="Inference_and_entailment">Inference and entailment</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Formal_system&amp;action=edit&amp;section=3" title="Edit section: Inference and entailment">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p>The <a href="/wiki/Entailment" class="mw-redirect" title="Entailment">entailment</a> of the system by its logical foundation is what distinguishes a formal system from others which may have some basis in an abstract model. Often the formal system will be the basis for or even identified with a larger theory or field (e.g. <a href="/wiki/Euclidean_geometry" title="Euclidean geometry">Euclidean geometry</a>) consistent with the usage in modern mathematics such as <a href="/wiki/Model_theory" title="Model theory">model theory</a>.<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="This section doesn't really do a group job stating what an entailment actually is. (September 2017)">clarification needed</span></a></i>]</sup> </p> 
   <h4><span class="mw-headline" id="Formal_language">Formal language</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Formal_system&amp;action=edit&amp;section=4" title="Edit section: Formal language">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
   <div role="note" class="hatnote navigation-not-searchable">
    Main articles: 
    <a href="/wiki/Formal_language" title="Formal language">Formal language</a> and 
    <a href="/wiki/Formal_grammar" title="Formal grammar">formal grammar</a>
   </div> 
   <p>A <a href="/wiki/Formal_language" title="Formal language">formal language</a> is a language that is defined by a formal system. Like languages in <a href="/wiki/Linguistics" title="Linguistics">linguistics</a>, formal languages generally have two aspects: </p> 
   <ul>
    <li>the <a href="/wiki/Syntax" title="Syntax">syntax</a> of a language is what the language looks like (more formally: the set of possible expressions that are valid utterances in the language) studied in <a href="/wiki/Formal_language_theory" class="mw-redirect" title="Formal language theory">formal language theory</a></li> 
    <li>the <a href="/wiki/Semantics" title="Semantics">semantics</a> of a language are what the utterances of the language mean (which is formalized in various ways, depending on the type of language in question)</li>
   </ul> 
   <p>In <a href="/wiki/Computer_science" title="Computer science">computer science</a> and <a href="/wiki/Linguistics" title="Linguistics">linguistics</a> a formal grammar is a precise description of a <a href="/wiki/Formal_language" title="Formal language">formal language</a>: a <a href="/wiki/Set_(mathematics)" title="Set (mathematics)">set</a> of <a href="/wiki/String_(computer_science)" title="String (computer science)">strings</a>. The two main categories of formal grammar are that of <a href="/wiki/Generative_grammar" title="Generative grammar">generative grammars</a>, which are sets of rules for how strings in a language can be generated, and that of <a href="/wiki/Analytic_grammar" class="mw-redirect" title="Analytic grammar">analytic grammars</a> (or reductive grammar,<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup><sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup>) which are sets of rules for how a string can be analyzed to determine whether it is a member of the language. In short, an analytic grammar describes how to <i>recognize</i> when strings are members in the set, whereas a generative grammar describes how to <i>write</i> only those strings in the set. </p> 
   <h4><span class="mw-headline" id="Logical_system">Logical system</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Formal_system&amp;action=edit&amp;section=5" title="Edit section: Logical system">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
   <p>A <i>logical system</i> or, for short, a <i>logic</i>, is a formal system together with its <a href="/wiki/Semantics_of_logic" title="Semantics of logic">semantics</a>,<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Disputed_statement" class="mw-redirect" title="Wikipedia:Disputed statement"><span title="This claim has reliable sources with contradicting facts (June 2015)">disputed</span></a> <span class="metadata"> – <a href="/wiki/Talk:Formal_system#No_semantics_needed" title="Talk:Formal system">discuss</a></span></i>]</sup>. According to <a href="/wiki/Model_theory" title="Model theory">model-theoretic</a> <a href="/wiki/Interpretation_(logic)" title="Interpretation (logic)">interpretation</a>, the semantics of a logical system describe whether a well-formed formula is satisfied by a given structure. A structure that satisfies all the axioms of the formal system is known as a model of the logical system. A logical system is <a href="/wiki/Soundness" title="Soundness">sound</a> if each well-formed formula that can be inferred from the axioms is satisfied by every model of the logical system. Conversely, a logic system is <a href="/wiki/Completeness_(logic)" title="Completeness (logic)">complete</a> if each well-formed formula that is satisfied by every model of the logical system can be inferred from the axioms. </p> 
   <h4><span class="mw-headline" id="Deductive_inference">Deductive inference</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Formal_system&amp;action=edit&amp;section=6" title="Edit section: Deductive inference">edit</a><span class="mw-editsection-bracket">]</span></span></h4> 
   <p>A <i>deductive system</i>, also called a <b>deductive apparatus</b>, consists of the <a href="/wiki/Axiom" title="Axiom">axioms</a> (or <a href="/wiki/Axiom_schema" title="Axiom schema">axiom schemata</a>) and <a href="/wiki/Rules_of_inference" class="mw-redirect" title="Rules of inference">rules of inference</a> that can be used to <a href="/wiki/Formal_proof" title="Formal proof">derive</a> <a href="/wiki/Theorem" title="Theorem">theorems</a> of the system.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> </p>
   <p>Such deductive systems preserve <a href="/wiki/Deductive_reasoning" title="Deductive reasoning">deductive</a> qualities in the <a href="/wiki/Formula_(mathematical_logic)" class="mw-redirect" title="Formula (mathematical logic)">formulas</a> that are expressed in the system. Usually the quality we are concerned with is <a href="/wiki/Truth" title="Truth">truth</a> as opposed to falsehood. However, other <a href="/wiki/Modal_logic" title="Modal logic">modalities</a>, such as <a href="/wiki/Theory_of_justification" title="Theory of justification">justification</a> or <a href="/wiki/Belief" title="Belief">belief</a> may be preserved instead. </p>
   <p>In order to sustain its deductive integrity, a <i>deductive apparatus</i> must be definable without reference to any <a href="/wiki/Intended_interpretation" class="mw-redirect" title="Intended interpretation">intended interpretation</a> of the language. The aim is to ensure that each line of a <a href="/wiki/Mathematical_proof" title="Mathematical proof">derivation</a> is merely a <a href="/wiki/Syntactic_consequence" class="mw-redirect" title="Syntactic consequence">syntactic consequence</a> of the lines that precede it. There should be no element of any <a href="/wiki/Interpretation_(logic)" title="Interpretation (logic)">interpretation</a> of the language that gets involved with the deductive nature of the system. </p> 
   <h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Formal_system&amp;action=edit&amp;section=7" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>Early logic systems includes Indian logic of <a href="/wiki/P%C4%81%E1%B9%87ini" title="Pāṇini">Pāṇini</a>, syllogistic logic of Aristotle, propositional logic of Stoicism,￼￼ and Chinese logic of <a href="/wiki/Gongsun_Long" title="Gongsun Long">Gongsun Long</a> (c. 325–250 BCE) . In recent times, contributors include <a href="/wiki/George_Boole" title="George Boole">George Boole</a>, <a href="/wiki/Augustus_De_Morgan" title="Augustus De Morgan">Augustus De Morgan</a>, and <a href="/wiki/Gottlob_Frege" title="Gottlob Frege">Gottlob Frege</a>. <a href="/wiki/Mathematical_logic" title="Mathematical logic">Mathematical logic</a> was developed in 19th century <a href="/wiki/Europe" title="Europe">Europe</a>. </p>

    <a href="/wiki/Formal_proof" title="Formal proof">Formal proof</a>
   </div> 
   <p>Formal proofs are sequences of <a href="/wiki/Well-formed_formula" title="Well-formed formula">well-formed formulas</a> (or wff for short). For a wff to qualify as part of a proof, it might either be an <a href="/wiki/Axiom" title="Axiom">axiom</a> or be the product of applying an inference rule on previous wffs in the proof sequence. The last wff in the sequence is recognized as a <a href="/wiki/Theorem#Theorems_in_logic" title="Theorem">theorem</a>. </p>
   <p>The point of view that generating formal proofs is all there is to mathematics is often called <i><a href="/wiki/Philosophy_of_mathematics#Formalism" title="Philosophy of mathematics">formalism</a></i>. <a href="/wiki/David_Hilbert" title="David Hilbert">David Hilbert</a> founded <a href="/wiki/Metamathematics" title="Metamathematics">metamathematics</a> as a discipline for discussing formal systems. Any language that one uses to talk about a formal system is called a <i><a href="/wiki/Metalanguage" title="Metalanguage">metalanguage</a></i>. The metalanguage may be a natural language, or it may be partially formalized itself, but it is generally less completely formalized than the formal language component of the formal system under examination, which is then called the <i><a href="/wiki/Object_language" title="Object language">object language</a></i>, that is, the object of the discussion in question. </p>
   <p>Once a formal system is given, one can define the set of theorems which can be proved inside the formal system. This set consists of all wffs for which there is a proof. Thus all axioms are considered theorems. Unlike the grammar for wffs, there is no guarantee that there will be a <a href="/wiki/Decidability_(logic)" title="Decidability (logic)">decision procedure</a> for deciding whether a given wff is a theorem or not. The notion of <i>theorem</i> just defined should not be confused with <i>theorems about the formal system</i>, which, in order to avoid confusion, are usually called <a href="/wiki/Metatheorem" title="Metatheorem">metatheorems</a>. </p>
