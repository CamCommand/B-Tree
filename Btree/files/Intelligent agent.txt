   <p>In <a href="/wiki/Artificial_intelligence" title="Artificial intelligence">artificial intelligence</a>, an <b>intelligent agent</b> (<b>IA</b>) refers to an <a href="/wiki/Autonomous" class="mw-redirect" title="Autonomous">autonomous</a> entity which acts, directing its activity towards achieving goals (i.e. it is an <a href="/wiki/Software_agent" title="Software agent">agent</a>), upon an <a href="/wiki/Environment_(biophysical)" class="mw-redirect" title="Environment (biophysical)">environment</a> using observation through sensors and consequent actuators (i.e. it is intelligent).<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2019)">citation needed</span></a></i>]</sup> Intelligent agents may also <a href="/wiki/Machine_learning" title="Machine learning">learn</a> or use <a href="/wiki/Knowledge_representation" class="mw-redirect" title="Knowledge representation">knowledge</a> to achieve their goals. They may be very simple or <a href="/wiki/Complexity" title="Complexity">very complex</a>. A reflex machine, such as a <a href="/wiki/Thermostat" title="Thermostat">thermostat</a>, is considered an example of an intelligent agent.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> </p>

   <p>Intelligent agents are often described schematically as an abstract functional system similar to a computer program. For this reason, intelligent agents are sometimes called <b>abstract intelligent agents</b> (<b>AIA</b>) to distinguish them from their real world implementations as computer systems, biological systems, or organizations. Some definitions of intelligent agents emphasize their autonomy, and so prefer the term <b>autonomous intelligent agents</b>. Still others (notably <a href="#CITEREFRussellNorvig2003">Russell &amp; Norvig (2003)</a>) considered goal-directed behavior as the essence of intelligence and so prefer a term borrowed from <a href="/wiki/Economics" title="Economics">economics</a>, "<a href="/wiki/Rational_agent" title="Rational agent">rational agent</a>". </p>
   <p>Intelligent agents in artificial intelligence are closely related to <a href="/wiki/Agent_(economics)" title="Agent (economics)">agents</a> in <a href="/wiki/Agent-based_computational_economics" title="Agent-based computational economics">economics</a>, and versions of the intelligent agent paradigm are studied in <a href="/wiki/Cognitive_science" title="Cognitive science">cognitive science</a>, <a href="/wiki/Ethic" class="mw-redirect" title="Ethic">ethics</a>, the philosophy of <a href="/wiki/Practical_reason" title="Practical reason">practical reason</a>, as well as in many <a href="/wiki/Interdisciplinary" class="mw-redirect" title="Interdisciplinary">interdisciplinary</a> <a href="/wiki/Socio-cognitive" title="Socio-cognitive">socio-cognitive</a> <a href="/wiki/Scientific_modelling" title="Scientific modelling">modeling</a> and computer <a href="/wiki/Social_simulation" title="Social simulation">social simulations</a>. </p>
   <p>Intelligent agents are also closely related to <a href="/wiki/Software_agent" title="Software agent">software agents</a> (an autonomous computer program that carries out tasks on behalf of users). In <a href="/wiki/Computer_science" title="Computer science">computer science</a>, the term <i>intelligent agent</i> may be used to refer to a software agent that has some intelligence, regardless if it is not a rational agent by Russell and Norvig's definition. For example, autonomous programs used for operator assistance or <a href="/wiki/Data_mining" title="Data mining">data mining</a> (sometimes referred to as <i><a href="/wiki/Software_agent" title="Software agent">bots</a></i>) are also called "intelligent agents". </p>

   <h2><span class="mw-headline" id="A_variety_of_definitions">A variety of definitions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intelligent_agent&amp;action=edit&amp;section=1" title="Edit section: A variety of definitions">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>Intelligent agents have been defined in many different ways.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> According to <a href="/w/index.php?title=Nikola_Kasabov&amp;action=edit&amp;redlink=1" class="new" title="Nikola Kasabov (page does not exist)">Nikola Kasabov</a><sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> AI systems should exhibit the following characteristics: </p> 
   <ul>
    <li>Accommodate new <a href="/wiki/Problem_solving" title="Problem solving">problem solving</a> rules incrementally</li> 
    <li>Adapt <a href="/wiki/Online" class="mw-redirect" title="Online">online</a> and in <a href="/wiki/Real-time_computing" title="Real-time computing">real time</a></li> 
    <li>Are able to analyze <a href="/wiki/Self-concept" title="Self-concept">themselves</a> in terms of behavior, error and success.</li> 
    <li>Learn and improve through interaction with the environment (<a href="/wiki/Embodied_agent" title="Embodied agent">embodiment</a>)</li> 
    <li>Learn quickly from large amounts of <a href="/wiki/Data_(computing)" title="Data (computing)">data</a></li> 
    <li>Have memory-based exemplar <a href="/wiki/Computer_storage" class="mw-redirect" title="Computer storage">storage</a> and retrieval capacities</li> 
    <li>Have <a href="/wiki/Parameter_(computer_science)" class="mw-redirect" title="Parameter (computer science)">parameters</a> to represent short and long term memory, age, forgetting, etc.</li>
   </ul> 
   <h2><span class="mw-headline" id="Structure_of_agents">Structure of agents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intelligent_agent&amp;action=edit&amp;section=2" title="Edit section: Structure of agents">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

   <p>Agent function is an abstract concept as it could incorporate various principles of decision making like calculation of <a href="/wiki/Utility" title="Utility">utility</a> of individual options, deduction over logic rules, <a href="/wiki/Fuzzy_logic" title="Fuzzy logic">fuzzy logic</a>, etc.<sup id="cite_ref-Salamon2011_5-0" class="reference"><a href="#cite_note-Salamon2011-5">[5]</a></sup> </p>
   <p>The <a href="/w/index.php?title=Program_agent&amp;action=edit&amp;redlink=1" class="new" title="Program agent (page does not exist)">program agent</a>, instead, maps every possible percept to an action<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2017)">citation needed</span></a></i>]</sup>. </p>
   <p>We use the term percept to refer to the agent's perceptional inputs at any given instant. In the following figures an agent is anything that can be viewed as perceiving its environment through sensors and acting upon that environment through actuators. </p> 
   <h2><span class="mw-headline" id="Architectures">Architectures</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intelligent_agent&amp;action=edit&amp;section=3" title="Edit section: Architectures">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p><a href="#CITEREFWeiss2013">Weiss (2013)</a> defines four classes of agents: </p> 
   <ul>
    <li>Logic-based agents – in which the decision about what action to perform is made via logical deduction;</li> 
    <li>Reactive agents – in which decision making is implemented in some form of direct mapping from situation to action;</li> 
    <li>Belief-desire-intention agents – in which decision making depends upon the manipulation of data structures representing the beliefs, desires, and intentions of the agent; and finally,</li> 
    <li>Layered architectures – in which decision making is realized via various software layers, each of which is more or less explicitly reasoning about the environment at different levels of abstraction.</li>
   </ul> 
   <p>Generally, an agent can be constructed by separating the body into the sensors and actuators, and so that it operates with a complex perception system that takes the description of the world as input for a controller and outputs commands to the actuator. However, a hierarchy of controller layers is often necessary to balance the immediate reaction desired for low-level tasks and the slow reasoning about complex, high-level goals.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> </p>

   <p><a href="#CITEREFRussellNorvig2003">Russell &amp; Norvig (2003)</a> group agents into five classes based on their degree of perceived intelligence and capability:<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup> </p> 
   <ol>
    <li>simple reflex agents</li> 
    <li>model-based reflex agents</li> 
    <li>goal-based agents</li> 
    <li>utility-based agents</li> 
    <li>learning agents</li>
   </ol> 
   <h3><span class="mw-headline" id="Simple_reflex_agents">Simple reflex agents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intelligent_agent&amp;action=edit&amp;section=5" title="Edit section: Simple reflex agents">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p>Simple reflex agents act only on the basis of the current percept, ignoring the rest of the percept history. The agent function is based on the <i>condition-action rule</i>: "if condition, then action". </p>
   <p>This agent function only succeeds when the environment is fully observable. Some reflex agents can also contain information on their current state which allows them to disregard conditions whose actuators are already triggered. </p>
   <p>Infinite loops are often unavoidable for simple reflex agents operating in partially observable environments. Note: If the agent can randomize its actions, it may be possible to escape from infinite loops. </p> 
   <h3><span class="mw-headline" id="Model-based_reflex_agents">Model-based reflex agents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intelligent_agent&amp;action=edit&amp;section=6" title="Edit section: Model-based reflex agents">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p>A model-based agent can handle partially observable environments. Its current state is stored inside the agent maintaining some kind of structure which describes the part of the world which cannot be seen. This knowledge about "how the world works" is called a model of the world, hence the name "model-based agent". </p>
   <p>A model-based reflex agent should maintain some sort of <a href="/wiki/Mental_model" title="Mental model">internal model</a> that depends on the percept history and thereby reflects at least some of the unobserved aspects of the current state. Percept history and impact of action on the environment can be determined by using internal model. It then chooses an action in the same way as reflex agent. </p>
   <p>An agent may also use models to describe and predict the behaviors of other agents in the environment.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> </p> 
   <h3><span class="mw-headline" id="Goal-based_agents">Goal-based agents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intelligent_agent&amp;action=edit&amp;section=7" title="Edit section: Goal-based agents">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p>Goal-based agents further expand on the capabilities of the model-based agents, by using "goal" information. Goal information describes situations that are desirable. This allows the agent a way to choose among multiple possibilities, selecting the one which reaches a goal state. Search and <a href="/wiki/Automated_planning_and_scheduling" title="Automated planning and scheduling">planning</a> are the subfields of artificial intelligence devoted to finding action sequences that achieve the agent's goals. </p> 
   <h3><span class="mw-headline" id="Utility-based_agents">Utility-based agents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intelligent_agent&amp;action=edit&amp;section=8" title="Edit section: Utility-based agents">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p>Goal-based agents only distinguish between goal states and non-goal states. It is possible to define a measure of how desirable a particular state is. This measure can be obtained through the use of a <i>utility function</i> which maps a state to a measure of the utility of the state. A more general performance measure should allow a <a href="/wiki/Comparison_(computer_programming)" class="mw-redirect" title="Comparison (computer programming)">comparison</a> of different world states according to exactly how happy they would make the agent. The term utility can be used to describe how "happy" the agent is. </p>
   <p>A rational utility-based agent chooses the action that maximizes the expected utility of the action outcomes - that is, what the agent expects to derive, on average, given the probabilities and utilities of each outcome. A utility-based agent has to model and keep track of its environment, tasks that have involved a great deal of research on perception, representation, reasoning, and learning. </p> 
   <h3><span class="mw-headline" id="Learning_agents">Learning agents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intelligent_agent&amp;action=edit&amp;section=9" title="Edit section: Learning agents">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p>Learning has the advantage that it allows the agents to initially operate in unknown environments and to become more competent than its initial knowledge alone might allow. The most important distinction is between the "learning element", which is responsible for making improvements, and the "performance element", which is responsible for selecting external actions. </p>
   <p>The learning element uses feedback from the "critic" on how the agent is doing and determines how the performance element should be modified to do better in the future. The performance element is what we have previously considered to be the entire agent: it takes in percepts and decides on actions. </p>
   <p>The last component of the learning agent is the "problem generator". It is responsible for suggesting actions that will lead to new and informative experiences. </p> 
   <h2><span class="mw-headline" id="Hierarchies_of_agents">Hierarchies of agents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Intelligent_agent&amp;action=edit&amp;section=10" title="Edit section: Hierarchies of agents">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <div role="note" class="hatnote navigation-not-searchable">
    Main article: 
    <a href="/wiki/Multi-agent_system" title="Multi-agent system">Multi-agent system</a>
   </div> 
   <p>To actively perform their <a href="/wiki/Function_(engineering)" title="Function (engineering)">functions</a>, Intelligent Agents today are normally gathered in a hierarchical structure containing many “sub-agents”. Intelligent sub-agents process and perform lower level functions. Taken together, the intelligent agent and sub-agents create a complete system that can accomplish difficult tasks or goals with behaviors and responses that display a form of intelligence. </p>
