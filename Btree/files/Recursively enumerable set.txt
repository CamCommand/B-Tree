   <p>In <a href="/wiki/Computability_theory" title="Computability theory">computability theory</a>, traditionally called recursion theory, a set <i>S</i> of <a href="/wiki/Natural_numbers" class="mw-redirect" title="Natural numbers">natural numbers</a> is called <b>recursively enumerable</b>, <b>computably enumerable</b>, <b>semidecidable</b>, <b>provable</b> or <b>Turing-recognizable</b> if: </p> 
   <ul>
    <li>There is an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> such that the set of input numbers for which the algorithm halts is exactly <i>S</i>.</li>
   </ul> 
   <p>Or, equivalently, </p> 
   <ul>
    <li>There is an algorithm that <a href="/wiki/Enumeration" title="Enumeration">enumerates</a> the members of <i>S</i>. That means that its output is simply a list of the members of <i>S</i>: <i>s</i><sub>1</sub>, <i>s</i><sub>2</sub>, <i>s</i><sub>3</sub>, ... . If necessary, this algorithm may run forever.</li>
   </ul> 
   <p>The first condition suggests why the term <i>semidecidable</i> is sometimes used; the second suggests why <i>computably enumerable</i> is used. The abbreviations <b>r.e.</b> and <b>c.e.</b> are often used, even in print, instead of the full phrase. </p>
   <p>In <a href="/wiki/Computational_complexity_theory" title="Computational complexity theory">computational complexity theory</a>, the <a href="/wiki/Complexity_class" title="Complexity class">complexity class</a> containing all recursively enumerable sets is <a href="/wiki/RE_(complexity)" title="RE (complexity)">RE</a>. In <a href="/wiki/Recursion_theory" class="mw-redirect" title="Recursion theory">recursion theory</a>, the <a href="/wiki/Lattice_(order)" title="Lattice (order)">lattice</a> of r.e. sets under inclusion is denoted <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;">

   <h2><span class="mw-headline" id="Formal_definition">Formal definition</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Recursively_enumerable_set&amp;action=edit&amp;section=1" title="Edit section: Formal definition">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>A set <i>S</i> of natural numbers is called <b>recursively enumerable</b> if there is a <a href="/wiki/Partial_recursive_function" class="mw-redirect" title="Partial recursive function">partial recursive function</a> whose <a href="/wiki/Domain_(mathematics)" class="mw-redirect" title="Domain (mathematics)">domain</a> is exactly <i>S</i>, meaning that the function is defined if and only if its input is a member of <i>S</i>. </p>

   <p>The equivalence of semidecidability and enumerability can be obtained by the technique of <a href="/wiki/Dovetailing_(computer_science)" title="Dovetailing (computer science)">dovetailing</a>. </p>
   <p>The Diophantine characterizations of a recursively enumerable set, while not as straightforward or intuitive as the first definitions, were found by <a href="/wiki/Yuri_Matiyasevich" title="Yuri Matiyasevich">Yuri Matiyasevich</a> as part of the negative solution to <a href="/wiki/Hilbert%27s_tenth_problem" title="Hilbert's tenth problem">Hilbert's Tenth Problem</a>. Diophantine sets predate recursion theory and are therefore historically the first way to describe these sets (although this equivalence was only remarked more than three decades after the introduction of recursively enumerable sets). The number of bound variables in the above definition of the <a href="/wiki/Diophantine_set" title="Diophantine set">Diophantine set</a> is the best known so far; it might be that a lower number can be used to define all diophantine sets. </p>

   <h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Recursively_enumerable_set&amp;action=edit&amp;section=3" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <ul>
    <li>Every <a href="/wiki/Recursive_set" title="Recursive set">recursive set</a> is recursively enumerable, but it is not true that every recursively enumerable set is recursive. For recursive sets, the algorithm must also say if an input is <i>not</i> in the set – this is not required of recursively enumerable sets.</li> 
    <li>A <a href="/wiki/Recursively_enumerable_language" title="Recursively enumerable language">recursively enumerable language</a> is a recursively enumerable subset of a <a href="/wiki/Formal_language" title="Formal language">formal language</a>.</li> 
    <li>The set of all provable sentences in an effectively presented axiomatic system is a recursively enumerable set.</li> 
    <li><a href="/wiki/Matiyasevich%27s_theorem" class="mw-redirect" title="Matiyasevich's theorem">Matiyasevich's theorem</a> states that every recursively enumerable set is a <a href="/wiki/Diophantine_set" title="Diophantine set">Diophantine set</a> (the converse is trivially true).</li> 
    <li>The <a href="/wiki/Simple_set" title="Simple set">simple sets</a> are recursively enumerable but not recursive.</li> 
    <li>The <a href="/wiki/Creative_set" class="mw-redirect" title="Creative set">creative sets</a> are recursively enumerable but not recursive.</li> 
    <li>Any <a href="/wiki/Productive_set" class="mw-redirect" title="Productive set">productive set</a> is <b>not</b> recursively enumerable.</li>

   <h2><span class="mw-headline" id="Properties">Properties</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Recursively_enumerable_set&amp;action=edit&amp;section=4" title="Edit section: Properties">edit</a><span class="mw-editsection-bracket">]</span></span></h2>

   <p>A set <i>A</i> is <a href="/wiki/Recursive_set" title="Recursive set">recursive</a> (synonym: computable) if and only if both <i>A</i> and the complement of <i>A</i> are recursively enumerable.    <p>Some pairs of recursively enumerable sets are <a href="/wiki/Effectively_separable" class="mw-redirect" title="Effectively separable">effectively separable</a> and some are not. </p> 
   <h2><span class="mw-headline" id="Remarks">Remarks</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Recursively_enumerable_set&amp;action=edit&amp;section=5" title="Edit section: Remarks">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>According to the <a href="/wiki/Church%E2%80%93Turing_thesis" title="Church–Turing thesis">Church–Turing thesis</a>, any effectively calculable function is calculable by a <a href="/wiki/Turing_machine" title="Turing machine">Turing machine</a>, and thus a set <i>S</i> is recursively enumerable if and only if there is some <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> which yields an enumeration of <i>S</i>. This cannot be taken as a formal definition, however, because the Church–Turing thesis is an informal conjecture rather than a formal axiom. </p>
   <p>The definition of a recursively enumerable set as the <i>domain</i> of a partial function, rather than the <i>range</i> of a total recursive function, is common in contemporary texts. This choice is motivated by the fact that in generalized recursion theories, such as <a href="/wiki/Alpha_recursion_theory" title="Alpha recursion theory">α-recursion theory</a>, the definition corresponding to domains has been found to be more natural. Other texts use the definition in terms of enumerations, which is equivalent for recursively enumerable sets. </p>
