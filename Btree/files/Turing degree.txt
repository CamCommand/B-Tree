   <h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_degree&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>The concept of Turing degree is fundamental in <a href="/wiki/Computability_theory" title="Computability theory">computability theory</a>, where sets of natural numbers are often regarded as <a href="/wiki/Decision_problem" title="Decision problem">decision problems</a>. The Turing degree of a set is a measure of how difficult it is to solve the decision problem associated with the set, that is, to determine whether an arbitrary number is in the given set. </p>
   <p>Two sets are <b>Turing equivalent</b> if they have the same level of unsolvability; each Turing degree is a collection of Turing equivalent sets, so that two sets are in different Turing degrees exactly when they are not Turing equivalent. Furthermore, the Turing degrees are <a href="/wiki/Partial_order" class="mw-redirect" title="Partial order">partially ordered</a> so that if the Turing degree of a set <i>X</i> is less than the Turing degree of a set <i>Y</i> then any (noncomputable) procedure that correctly decides whether numbers are in <i>Y</i> can be effectively converted to a procedure that correctly decides whether numbers are in <i>X</i>. It is in this sense that the Turing degree of a set corresponds to its level of algorithmic unsolvability. </p>
   <p>The Turing degrees were introduced by <a href="/wiki/Emil_Leon_Post" title="Emil Leon Post">Emil Leon Post</a> (1944), and many fundamental results were established by <a href="/wiki/Stephen_Cole_Kleene" title="Stephen Cole Kleene">Stephen Cole Kleene</a> and Post (1954). The Turing degrees have been an area of intense research since then. Many proofs in the area make use of a proof technique known as the <b>priority method</b>. </p> 
   <h2><span class="mw-headline" id="Turing_equivalence">Turing equivalence</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_degree&amp;action=edit&amp;section=2" title="Edit section: Turing equivalence">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <div role="note" class="hatnote navigation-not-searchable">
    Main article: 
    <a href="/wiki/Turing_reduction" title="Turing reduction">Turing reduction</a>
   </div> 
   <p>For the rest of this article, the word <i>set</i> will refer to a set of natural numbers. A set <i>X</i> is said to be <b><a href="/wiki/Turing_reducible" class="mw-redirect" title="Turing reducible">Turing reducible</a></b> to a set <i>Y</i> if there is an <a href="/wiki/Oracle_Turing_machine" class="mw-redirect" title="Oracle Turing machine">oracle Turing machine</a> that decides membership in <i>X</i> when given an oracle for membership in <i>Y</i>. The notation <i>X</i> ≤<sub>T</sub> <i>Y</i> indicates that <i>X</i> is Turing reducible to <i>Y</i>. </p>
   <p>Two sets <i>X</i> and <i>Y</i> are defined to be <b>Turing equivalent</b> if <i>X</i> is Turing reducible to <i>Y</i> and <i>Y</i> is Turing reducible to <i>X</i>. The notation <i>X</i> ≡<sub>T</sub> <i>Y</i> indicates that <i>X</i> and <i>Y</i> are Turing equivalent. The relation ≡<sub>T</sub> can be seen to be an <a href="/wiki/Equivalence_relation" title="Equivalence relation">equivalence relation</a>, which means that for all sets <i>X</i>, <i>Y</i>, and <i>Z</i>: </p> 
   <ul>
    <li><i>X</i> ≡<sub>T</sub> <i>X</i></li> 
    <li><i>X</i> ≡<sub>T</sub> <i>Y</i> implies <i>Y</i> ≡<sub>T</sub> <i>X</i></li> 
    <li>If <i>X</i> ≡<sub>T</sub> <i>Y</i> and <i>Y</i> ≡<sub>T</sub> <i>Z</i> then <i>X</i> ≡<sub>T</sub> <i>Z</i>.</li>
   </ul>

The notation [<i>X</i>] denotes the equivalence class containing a set <i>X</i>. A key example is <b>0</b>′, the degree of the <a href="/wiki/Halting_problem" title="Halting problem">halting problem</a>. </p>

   <h2><span class="mw-headline" id="Structure_of_the_Turing_degrees">Structure of the Turing degrees</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_degree&amp;action=edit&amp;section=4" title="Edit section: Structure of the Turing degrees">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>A great deal of research has been conducted into the structure of the Turing degrees. The following survey lists only some of the many known results. One general conclusion that can be drawn from the research is that the structure of the Turing degrees is extremely complicated. </p> 
   <h3><span class="mw-headline" id="Order_properties">Order properties</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_degree&amp;action=edit&amp;section=5" title="Edit section: Order properties">edit</a><span class="mw-editsection-bracket">]</span></span></h3>

    <li>For every nonzero degree <b>a</b> there is a degree <b>b</b> incomparable with <b>a</b>.</li>

   <h3><span class="mw-headline" id="Properties_involving_the_jump">Properties involving the jump</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_degree&amp;action=edit&amp;section=6" title="Edit section: Properties involving the jump">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <ul>
    <li>For every degree <b>a</b> there is a degree strictly between <b>a</b> and <b>a′</b>. In fact, there is a countable sequence of pairwise incomparable degrees between <b>a</b> and <b>a′</b>.</li> 
    <li>A degree <b>a</b> is of the form <b>b′</b> if and only if <b>0′</b> ≤ <b>a</b>.</li> 
    <li>For any degree <b>a</b> there is a degree <b>b</b> such that <b>a</b> &lt; <b>b</b> and <b>b′</b> = <b>a′</b>; such a degree <b>b</b> is called <i>low</i> relative to <b>a</b>.</li> 
    <li>There is an infinite sequence <b>a</b><sub><i>i</i></sub> of degrees such that <b>a</b>′<sub><i>i</i>+1</sub> ≤ <b>a</b><sub><i>i</i></sub> for each <i>i</i>.</li>
   </ul>

   <ul>
    <li>Simpson (1977) showed that the first-order theory of <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;">

   <h2><span class="mw-headline" id="Structure_of_the_r.e._Turing_degrees">Structure of the r.e. Turing degrees</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_degree&amp;action=edit&amp;section=8" title="Edit section: Structure of the r.e. Turing degrees">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>A degree is called r.e. (recursively enumerable) if it contains a <a href="/wiki/Recursively_enumerable_set" title="Recursively enumerable set">recursively enumerable set</a>. Every r.e. degree is less than or equal to <b>0′</b> but not every degree less than <b>0′</b> is an r.e. degree. </p> 
   <ul>
    <li>(<a href="/wiki/Gerald_Sacks" title="Gerald Sacks">G. E. Sacks</a>, 1964) The r.e degrees are dense; between any two r.e. degrees there is a third r.e degree.</li> 
    <li>(A. H. Lachlan, 1966a and C. E. M. Yates, 1966) There are two r.e. degrees with no greatest lower bound in the r.e. degrees.</li> 
    <li>(A. H. Lachlan, 1966a and C. E. M. Yates, 1966) There is a pair of nonzero r.e. degrees whose greatest lower bound is <b>0</b>.</li> 
    <li>(S. K. Thomason, 1971) Every finite distributive lattice can be embedded into the r.e. degrees. In fact, the countable atomless Boolean algebra can be embedded in a manner that preserves suprema and infima.</li> 
    <li>(A. H. Lachlan and <a href="/wiki/Robert_I._Soare" title="Robert I. Soare">R. I. Soare</a>, 1980) Not all finite lattices can be embedded in the r.e. degrees (via an embedding that preserves suprema and infima). The following particular <a href="/wiki/Lattice_(order)" title="Lattice (order)">lattice</a> cannot be embedded in the r.e. degrees:</li>
   </ul> 
   <dl>
    <dd>
     <dl>
      <dd>
       <a href="/wiki/File:Rehasse.png" class="image"><img alt="Rehasse.png" src="//upload.wikimedia.org/wikipedia/commons/e/ea/Rehasse.png" decoding="async" width="78" height="141" data-file-width="78" data-file-height="141"></a>
      </dd>
     </dl>
    </dd>
   </dl> 
   <ul>
    <li>(A. H. Lachlan, 1966b) There is no pair of r.e. degrees whose greatest lower bound is <b>0</b> and whose least upper bound is <b>0′</b>. This result is informally called the <i>nondiamond theorem</i>.</li> 
    <li>(<a href="/wiki/Leo_Harrington" title="Leo Harrington">L. A. Harrington</a> and <a href="/wiki/Theodore_Slaman" title="Theodore Slaman">T. A. Slaman</a>, see Nies, Shore, and Slaman (1998)) The first-order theory of the r.e. degrees in the language 〈 <b>0</b>, ≤, = 〉 is many-one equivalent to the theory of true first order arithmetic.</li>
   </ul> 
   <h2><span id="Post.27s_problem_and_the_priority_method"></span><span class="mw-headline" id="Post's_problem_and_the_priority_method">Post's problem and the priority method</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_degree&amp;action=edit&amp;section=9" title="Edit section: Post's problem and the priority method">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <div role="note" class="hatnote navigation-not-searchable">
    "Post's problem" redirects here. For the other "Post's problem", see 
    <a href="/wiki/Post%27s_correspondence_problem" class="mw-redirect" title="Post's correspondence problem">Post's correspondence problem</a>.
   </div> 
   <p><a href="/wiki/Emil_Post" class="mw-redirect" title="Emil Post">Emil Post</a> studied the r.e.&nbsp;Turing degrees and asked whether there is any r.e. degree strictly between <b>0</b> and <b>0′</b>. The problem of constructing such a degree (or showing that none exist) became known as <b>Post's problem</b>. This problem was solved independently by Friedberg and <a href="/wiki/Albert_Muchnik" title="Albert Muchnik">Muchnik</a> in the 1950s, who showed that these intermediate r.e. degrees do exist. Their proofs each developed the same new method for constructing r.e. degrees which came to be known as the <b>priority method</b>. The priority method is now the main technique for establishing results about r.e. sets. </p>
   <p>The idea of the priority method for constructing a r.e. set <i>X</i> is to list a countable sequence of <i>requirements</i> that <i>X</i> must satisfy. For example, to construct a r.e. set <i>X</i> between <b>0</b> and <b>0′</b> it is enough to satisfy the requirements <i>A<sub>e</sub></i> and <i>B<sub>e</sub></i> for each natural number <i>e</i>, where <i>A<sub>e</sub></i> requires that the oracle machine with index <i>e</i> does not compute 0′ from <i>X</i> and <i>B<sub>e</sub></i> requires that the Turing machine with index <i>e</i> (and no oracle) does not compute <i>X</i>. These requirements are put into a <i>priority ordering</i>, which is an explicit bijection of the requirements and the natural numbers. The proof proceeds inductively with one stage for each natural number; these stages can be thought of as steps of time during which the set <i>X</i> is enumerated. At each stage, numbers may be put into <i>X</i> or forever prevented from entering <i>X</i> in an attempt to <i>satisfy</i> requirements (that is, force them to hold once all of <i>X</i> has been enumerated). Sometimes, a number can be enumerated into <i>X</i> to satisfy one requirement but doing this would cause a previously satisfied requirement to become unsatisfied (that is, to be <i>injured</i>). The priority order on requirements is used to determine which requirement to satisfy in this case. The informal idea is that if a requirement is injured then it will eventually stop being injured after all higher priority requirements have stopped being injured, although not every priority argument has this property. An argument must be made that the overall set <i>X</i> is r.e. and satisfies all the requirements. Priority arguments can be used to prove many facts about r.e. sets; the requirements used and the manner in which they are satisfied must be carefully chosen to produce the required result. </p>
