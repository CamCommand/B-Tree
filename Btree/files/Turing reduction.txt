  <div class="mw-parser-output">
   <p>In <a href="/wiki/Computability_theory" title="Computability theory">computability theory</a>, a <b>Turing reduction</b> (also known as a <b>Cook reduction</b>) from a problem <i>A</i> to a problem <i>B</i>, is a <a href="/wiki/Reduction_(complexity)" title="Reduction (complexity)">reduction</a> which solves <i>A</i>, assuming the solution to <i>B</i> is already known (Rogers 1967, Soare 1987). It can be understood as an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> that could be used to solve <i>A</i> if it had available to it a <a href="/wiki/Subroutine" title="Subroutine">subroutine</a> for solving <i>B</i>. More formally, a Turing reduction is a function computable by an <a href="/wiki/Oracle_machine" title="Oracle machine">oracle machine</a> with an oracle for <i>B</i>. Turing reductions can be applied to both <a href="/wiki/Decision_problem" title="Decision problem">decision problems</a> and <a href="/wiki/Function_problem" title="Function problem">function problems</a>. </p>
   <p>If a Turing reduction of <i>A</i> to <i>B</i> exists then every <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> for <i>B</i> can be used to produce an algorithm for <i>A</i>, by inserting the algorithm for <i>B</i> at each place where the oracle machine computing <i>A</i> queries the oracle for <i>B</i>. However, because the oracle machine may query the oracle a large number of times, the resulting algorithm may require more time asymptotically than either the algorithm for <i>B</i> or the oracle machine computing <i>A</i>, and may require as much space as both together. </p>
   <p>The first formal definition of relative computability, then called relative reducibility, was given by <a href="/wiki/Alan_Turing" title="Alan Turing">Alan Turing</a> in 1939 in terms of <a href="/wiki/Oracle_machine" title="Oracle machine">oracle machines</a>. Later in 1943 and 1952 <a href="/wiki/Stephen_Kleene" class="mw-redirect" title="Stephen Kleene">Stephen Kleene</a> defined an equivalent concept in terms of <a href="/wiki/Mu-recursive_function" class="mw-redirect" title="Mu-recursive function">recursive functions</a>. In 1944 <a href="/wiki/Emil_Post" class="mw-redirect" title="Emil Post">Emil Post</a> used the term "Turing reducibility" to refer to the concept. </p>
   <p>A <a href="/wiki/Polynomial-time_reduction" title="Polynomial-time reduction">polynomial-time</a> Turing reduction is known as a <b>Cook reduction</b>, after <a href="/wiki/Stephen_Cook" title="Stephen Cook">Stephen Cook</a>. </p>

   <h2><span class="mw-headline" id="The_use_of_a_reduction">The use of a reduction</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_reduction&amp;action=edit&amp;section=5" title="Edit section: The use of a reduction">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>Since every reduction from a set <i>B</i> to a set <i>A</i> has to determine whether a single element is in <i>A</i> in only finitely many steps, it can only make finitely many queries of membership in the set <i>B</i>. When the amount of information about the set <i>B</i> used to compute a single bit of <i>A</i> is discussed, this is made precise by the use function. Formally, the <i>use</i> of a reduction is the function that sends each natural number <i>n</i> to the largest natural number <i>m</i> whose membership in the set <i>B</i> was queried by the reduction while determining the membership of <i>n</i> in <i>A</i>. </p> 
   <h2><span class="mw-headline" id="Stronger_reductions">Stronger reductions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_reduction&amp;action=edit&amp;section=6" title="Edit section: Stronger reductions">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>There are two common ways of producing reductions stronger than Turing reducibility. The first way is to limit the number and manner of oracle queries. </p> 
   <ul>
    <li>A set <i>A</i> is <b><a href="/wiki/Many-one_reduction" title="Many-one reduction">many-one reducible</a></b> to <i>B</i> if there is a <a href="/wiki/Computable_function" title="Computable function">total computable function</a> <i>f</i> such that an element <i>n</i> is in <i>A</i> if and only if <i>f</i>(<i>n</i>) is in <i>B</i>. Such a function can be used to generate a Turing reduction (by computing <i>f</i>(<i>n</i>), querying the oracle, and then interpreting the result).</li> 
    <li>A <b><a href="/wiki/Truth_table_reduction" class="mw-redirect" title="Truth table reduction">truth table reduction</a></b> or a <b><a href="/wiki/Truth_table_reduction" class="mw-redirect" title="Truth table reduction">weak truth table reduction</a></b> must present all of its oracle queries at the same time. In a truth table reduction, the reduction also gives a boolean function (a <b>truth table</b>) which, when given the answers to the queries, will produce the final answer of the reduction. In a weak truth table reduction, the reduction uses the oracle answers as a basis for further computation depending on the given answers (but not using the oracle). Equivalently, a weak truth table reduction is one for which the use of the reduction is bounded by a computable function. For this reason, weak truth table reductions are sometimes called "bounded Turing" reductions.</li>
   </ul> 
   <p>The second way to produce a stronger reducibility notion is to limit the computational resources that the program implementing the Turing reduction may use. These limits on the <a href="/wiki/Computational_complexity_theory" title="Computational complexity theory">computational complexity</a> of the reduction are important when studying subrecursive classes such as <a href="/wiki/P_(complexity)" title="P (complexity)">P</a>. A set <i>A</i> is <b><a href="/wiki/Polynomial-time_reduction" title="Polynomial-time reduction">polynomial-time reducible</a></b> to a set <i>B</i> if there is a Turing reduction of <i>A</i> to <i>B</i> that runs in polynomial time. The concept of <b><a href="/wiki/Log-space_reduction" title="Log-space reduction">log-space reduction</a></b> is similar. </p>
   <p>These reductions are stronger in the sense that they provide a finer distinction into equivalence classes, and satisfy more restrictive requirements than Turing reductions. Consequently, such reductions are harder to find. There may be no way to build a many-one reduction from one set to another even when a Turing reduction for the same sets exists. </p> 
   <h2><span class="mw-headline" id="Weaker_reductions">Weaker reductions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_reduction&amp;action=edit&amp;section=7" title="Edit section: Weaker reductions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
