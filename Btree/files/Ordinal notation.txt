   <p>In <a href="/wiki/Mathematical_logic" title="Mathematical logic">mathematical logic</a> and <a href="/wiki/Set_theory" title="Set theory">set theory</a>, an <b>ordinal notation</b> is a partial function from the set of all finite sequences of symbols from a finite alphabet to a countable set of <a href="/wiki/Ordinal_number" title="Ordinal number">ordinals</a>, and a <a href="/wiki/G%C3%B6del_numbering" title="Gödel numbering">Gödel numbering</a> is a function from the set of well-formed formulae (a well-formed formula is a finite sequence of symbols on which the ordinal notation function is defined) of some formal language to the natural numbers. This associates each wff with a unique natural number, called its Gödel number. If a Gödel numbering is fixed, then the subset relation on the ordinals induces an ordering on well-formed formulae which in turn induces a well-ordering on the subset of natural numbers. A <b>recursive ordinal notation</b> must satisfy the following two additional properties: </p> 
   <ol>
    <li>the subset of natural numbers is a <a href="/wiki/Recursive_set" title="Recursive set">recursive set</a></li> 
    <li>the induced well-ordering on the subset of natural numbers is a recursive relation</li>
   </ol> 
   <p>There are many such schemes of ordinal notations, including schemes by <a href="/wiki/Wilhelm_Ackermann" title="Wilhelm Ackermann">Wilhelm Ackermann</a>, <a href="/wiki/Heinz_Bachmann" title="Heinz Bachmann">Heinz Bachmann</a>, Wilfried Buchholz, <a href="/wiki/Georg_Cantor" title="Georg Cantor">Georg Cantor</a>, <a href="/wiki/Solomon_Feferman" title="Solomon Feferman">Solomon Feferman</a>, Gerhard Jäger, Isles, Pfeiffer, Wolfram Pohlers, <a href="/wiki/Kurt_Sch%C3%BCtte" title="Kurt Schütte">Kurt Schütte</a>, <a href="/wiki/Gaisi_Takeuti" title="Gaisi Takeuti">Gaisi Takeuti</a> (called <b>ordinal diagrams</b>), <a href="/wiki/Oswald_Veblen" title="Oswald Veblen">Oswald Veblen</a>. <a href="/wiki/Stephen_Cole_Kleene" title="Stephen Cole Kleene">Stephen Cole Kleene</a> has a system of notations, called <a href="/wiki/Kleene%27s_O" title="Kleene's O">Kleene's O</a>, which includes ordinal notations but it is not as well behaved as the other systems described here. </p>
   <p>Usually one proceeds by defining several functions from ordinals to ordinals and representing each such function by a symbol. In many systems, such as Veblen's well known system, the functions are normal functions, that is, they are strictly increasing and continuous in at least one of their arguments, and increasing in other arguments. Another desirable property for such functions is that the value of the function is greater than each of its arguments, so that an ordinal is always being described in terms of smaller ordinals. There are several such desirable properties. Unfortunately, no one system can have all of them since they contradict each other. </p>

   <h2><span class="mw-headline" id="A_simplified_example_using_a_pairing_function">A simplified example using a pairing function</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordinal_notation&amp;action=edit&amp;section=1" title="Edit section: A simplified example using a pairing function">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>As usual, we must start off with a constant symbol for zero, "0", which we may consider to be a function of <a href="/wiki/Arity" title="Arity">arity</a> zero. This is necessary because there are no smaller ordinals in terms of which zero can be described. The most obvious next step would be to define a unary function, "S", which takes an ordinal to the smallest ordinal greater than it; in other words, S is the successor function. In combination with zero, successor allows one to name any natural number. </p>
   <p>The third function might be defined as one that maps each ordinal to the smallest ordinal that cannot yet be described with the above two functions and previous values of this function. This would map β to ω·β except when β is a fixed point of that function plus a finite number in which case one uses ω·(β+1). </p>
   <p>The fourth function would map α to ω<sup>ω</sup>·α except when α is a fixed point of that plus a finite number in which case one uses ω<sup>ω</sup>·(α+1). </p> 
   <h3><span id=".CE.BE-notation"></span><span class="mw-headline" id="ξ-notation">ξ-notation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordinal_notation&amp;action=edit&amp;section=2" title="Edit section: ξ-notation">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p>One could continue in this way, but it would give us an infinite number of functions. So instead let us merge the unary functions into a binary function. By <a href="/wiki/Transfinite_recursion" class="mw-redirect" title="Transfinite recursion">transfinite recursion</a> on α, we can use transfinite recursion on β to define ξ(α,β) = the smallest ordinal γ such that α &lt; γ and β &lt; γ and γ is not the value of ξ for any smaller α or for the same α with a smaller β. </p>
   <p>Thus, define ξ-notations as follows: </p> 
   <ul>
    <li>"0" is a ξ-notation for zero.</li> 
    <li>If "A" and "B" are replaced by ξ-notations for α and β in "ξAB", then the result is a ξ-notation for ξ(α,β).</li> 
    <li>There are no other ξ-notations.</li>
   </ul> 
   <p>ξ is defined for all pairs of ordinals and is one-to-one. It always gives values larger than its arguments and its <a href="/wiki/Range_(mathematics)" title="Range (mathematics)">range</a> is all ordinals other than 0 and the epsilon numbers (ε=ω<sup>ε</sup>). </p>
   <p>ξ(α,β)&lt;ξ(γ,δ) if and only if either (α=γ and β&lt;δ) or (α&lt;γ and β&lt;ξ(γ,δ)) or (α&gt;γ and ξ(α,β)≤δ). </p>

   <p>In general, ξ(0,β) = β+1. While ξ(1+α,β) = ω<sup>ω<sup>α</sup></sup>·(β+k) for k = 0 or 1 or 2 depending on special situations:<br> k = 2 if α is an epsilon number and β is finite.<br> Otherwise, k = 1 if β is a multiple of ω<sup>ω<sup>α+1</sup></sup> plus a finite number.<br> Otherwise, k = 0. </p>
   <p>The ξ-notations can be used to name any ordinal less than ε<sub>0</sub> with an alphabet of only two symbols ("0" and "ξ"). If these notations are extended by adding functions that enumerate epsilon numbers, then they will be able to name any ordinal less than the first epsilon number that cannot be named by the added functions. This last property, adding symbols within an initial segment of the ordinals gives names within that segment, is called repleteness (after <a href="/wiki/Solomon_Feferman" title="Solomon Feferman">Solomon Feferman</a>). </p> 
   <h2><span class="mw-headline" id="Systems_of_ordinal_notation">Systems of ordinal notation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordinal_notation&amp;action=edit&amp;section=3" title="Edit section: Systems of ordinal notation">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
   <p>There are many different systems for ordinal notation introduced by various authors. It is often quite hard to convert between the different systems. </p> 
   <h3><span class="mw-headline" id="Cantor">Cantor</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordinal_notation&amp;action=edit&amp;section=4" title="Edit section: Cantor">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <div role="note" class="hatnote navigation-not-searchable">
    Main article: 
    <a href="/wiki/Cantor_normal_form" class="mw-redirect" title="Cantor normal form">Cantor normal form</a>
   </div> 
   <p>"Exponential polynomials" in 0 and ω gives a system of ordinal notation for ordinals less than <a href="/wiki/Epsilon_zero" class="mw-redirect" title="Epsilon zero">epsilon zero</a>. There are many equivalent ways to write these; instead of exponential polynomials, one can use rooted trees, or nested parentheses, or the system described above. </p> 
   <h3><span class="mw-headline" id="Veblen">Veblen</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordinal_notation&amp;action=edit&amp;section=5" title="Edit section: Veblen">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <div role="note" class="hatnote navigation-not-searchable">
    Main article: 
    <a href="/wiki/Veblen_function" title="Veblen function">Veblen function</a>
   </div> 
   <p>The 2-variable Veblen functions (<a href="#CITEREFVeblen1908">Veblen 1908</a>) can be used to give a system of ordinal notation for ordinals less than the <a href="/wiki/Feferman-Schutte_ordinal" class="mw-redirect" title="Feferman-Schutte ordinal">Feferman-Schutte ordinal</a>. The Veblen functions in a finite or transfinite number of variables give systems of ordinal notations for ordinals less than the small and large <a href="/wiki/Veblen_ordinal_(disambiguation)" class="mw-redirect mw-disambig" title="Veblen ordinal (disambiguation)">Veblen ordinals</a>. </p> 
   <h3><span class="mw-headline" id="Ackermann">Ackermann</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordinal_notation&amp;action=edit&amp;section=6" title="Edit section: Ackermann">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p><a href="#CITEREFAckermann1951">Ackermann (1951)</a> described a system of ordinal notation rather weaker than the system described earlier by Veblen. The limit of his system is sometimes called the <a href="/wiki/Ackermann_ordinal" title="Ackermann ordinal">Ackermann ordinal</a>. </p> 
   <h3><span class="mw-headline" id="Bachmann">Bachmann</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordinal_notation&amp;action=edit&amp;section=7" title="Edit section: Bachmann">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p><a href="#CITEREFBachmann1950">Bachmann (1950)</a> introduced the key idea of using uncountable ordinals to produce new countable ordinals. His original system was rather cumbersome to use as it required choosing a special sequence converging to each ordinal. Later systems of notation introduced by Feferman and others avoided this complication. </p> 
   <h3><span id="Takeuti_.28ordinal_diagrams.29"></span><span class="mw-headline" id="Takeuti_(ordinal_diagrams)">Takeuti (ordinal diagrams)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordinal_notation&amp;action=edit&amp;section=8" title="Edit section: Takeuti (ordinal diagrams)">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p><a href="#CITEREFTakeuti1987">Takeuti (1987)</a> described a very powerful system of ordinal notation called "ordinal diagrams", which is hard to understand but was later simplified by Feferman. </p> 
   <h3><span id="Feferman.27s_.CE.B8_functions"></span><span class="mw-headline" id="Feferman's_θ_functions">Feferman's θ functions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordinal_notation&amp;action=edit&amp;section=9" title="Edit section: Feferman's θ functions">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <p>Feferman introduced theta functions, described in <a href="#CITEREFBuchholz1986">Buchholz (1986)</a> as follows. The function for an ordinal α, θ<sub>α</sub> is a function from ordinals to ordinals. Often θ<sub>α</sub>(β) is written as θαβ. The set <i>C</i>(α,β) is defined by induction on α to be the set of ordinals that can be generated from 0, <a href="/wiki/First_uncountable_ordinal" title="First uncountable ordinal">ω<sub>1</sub></a>, ω<sub>2</sub>, ..., ω<sub>ω</sub>, together with the ordinals less than β by the operations of ordinal addition and the functions θ<sub>ξ</sub> for ξ&lt;α. And the function θ<sub>γ</sub> is defined to be the function enumerating the ordinals δ with δ∉<i>C</i>(γ,δ). </p> 
   <h3><span class="mw-headline" id="Buchholz">Buchholz</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ordinal_notation&amp;action=edit&amp;section=10" title="Edit section: Buchholz">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
   <div role="note" class="hatnote navigation-not-searchable">

   <p><a href="#CITEREFBuchholz1986">Buchholz (1986)</a> described the following system of ordinal notation as a simplification of Feferman's theta functions. Define: </p> 
   <ul>
    <li>Ω<sub>ξ</sub> = ω<sub>ξ</sub> if ξ &gt; 0, Ω<sub>0</sub> = 1</li>
   </ul> 
   <p>The functions ψ<sub><i>v</i></sub>(α) for α an ordinal, <i>v</i> an ordinal at most ω, are defined by induction on α as follows: </p> 
   <ul>
    <li>ψ<sub><i>v</i></sub>(α) is the smallest ordinal not in <i>C</i><sub><i>v</i></sub>(α)</li>
   </ul> 
   <p>where <i>C</i><sub><i>v</i></sub>(α) is the smallest set such that </p> 
   <ul>
    <li><i>C</i><sub><i>v</i></sub>(α) contains all ordinals less than Ω<sub><i>v</i></sub></li> 
    <li><i>C</i><sub><i>v</i></sub>(α) is closed under ordinal addition</li> 
    <li><i>C</i><sub><i>v</i></sub>(α) is closed under the functions ψ<sub><i>u</i></sub> (for <i>u</i>≤ω) applied to arguments less than α.</li>
   </ul>

   <p><a href="#CITEREFKleene1938">Kleene (1938)</a> described a system of notation for all recursive ordinals (those less than the <a href="/wiki/Church%E2%80%93Kleene_ordinal" title="Church–Kleene ordinal">Church–Kleene ordinal</a>). It uses a subset of the <a href="/wiki/Natural_number" title="Natural number">natural numbers</a> instead of finite strings of symbols. Unfortunately, unlike the other systems described above there is in general no <a href="/wiki/Computable_function" title="Computable function">effective</a> way to tell whether some natural number represents an ordinal, or whether two numbers represent the same ordinal. However, one can effectively find notations that represent the ordinal sum, product, and power (see <a href="/wiki/Ordinal_arithmetic" title="Ordinal arithmetic">ordinal arithmetic</a>) of any two given notations in Kleene's <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;">
